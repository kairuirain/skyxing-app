# SkyXing 1.2.1 更新日志

> 发布性质：OTA v3 协议升级 + 自定义通知系统 + UI 重设（OTA v3 Protocol + Custom Notices + UI Refresh）
> 覆盖平台：Web · Android (Tauri) · Windows

1.2.1 引入 OTA v3 协议，新增管理员自定义通知模块，支持插入文本提示（如修复提醒）。APP 端首页进行 B 站风格 UI 重设，写文章入口全面覆盖。

## 一、OTA v3 自定义通知系统

- **协议升级**：`/check` 与 `/latest` 接口新增 `protocolVersion: 3` 与 `notices` 数组，旧客户端忽略未知字段即可向下兼容。
- **自定义通知**：管理员可通过 `PUT /updates/notice` 添加通知（指定平台、版本范围、标题、内容、操作按钮）。
- **通知过滤**：按 `platform` 和版本范围（`minVersion`/`maxVersion`）智能匹配，精准触达目标用户。
- **独立接口**：新增 `GET /updates/notice` 公共端点，旧客户端可单独拉取通知。
- **通知示例**：针对安卓端网页跳转 bug，可推送 "安卓端网页连接跳转出现了 bug，请手动访问 XXX" 提示。

## 二、APP 端 UI 重设（B 站风格）

- **B 站风格 Hero 区域**：粉蓝渐变背景 + 平台品牌展示 + 快捷入口（写文章 / 私信）。
- **写文章 FAB 修复**：安卓端 `bottom-20` 避免被底部导航遮挡，桌面端 `bottom-6`；空文章列表时显示"写第一篇"引导按钮。
- **OTA v3 通知条**：首页与"我的"页面均展示自定义通知（支持关闭、跳转操作）。
- **设置页优化**：调试 / 更新 / 关于软件分区清晰，更新通道切换即生效。

## 三、修复问题

- 修复安卓端 Tauri 应用没有"写文章"按钮的问题（FAB 浮动按钮 + hero 快捷入口 + 空状态引导）。
- 修复安卓端底部导航遮挡写文章按钮的问题（`bottom-20` 自适应）。
- 修复安卓端网页连接无法跳转至默认浏览器的问题（`@tauri-apps/plugin-shell` 的 `open` API）。
- 修复 `updates.js` 残留孤立代码导致 wrangler deploy 失败的问题。

## 四、性能优化（1.1.x 延续）

- 列表并行批量获取、评论索引 O(1)、边缘缓存头。
- 3 秒轻量轮询 + Visibility API 暂停 + 手动刷新按钮。
