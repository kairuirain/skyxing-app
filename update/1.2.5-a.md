# SkyXing 1.2.5-a 更新日志

> 发布性质：测试版（Bugfix）
> 覆盖平台：Web · Tauri APP (Windows/Android)

## 概要

1.2.5-a 修复了 Web 端非首页刷新 500 错误、Tauri 端文章链接跳转异常等问题，优化了文章渲染策略。

---

## 问题修复

### 1. 修复 Web 端路由刷新 500 错误

**问题**：主页 `/` 刷新正常，但在 `/article/xxx`、`/write` 等页面刷新或直接访问时返回 `Internal Server Error`（API 系统运行正常）。

**原因**：Cloudflare Worker 的 `app.get('/*', serveStatic(...))` 回退路由尝试加载 `public/index.html`，但该文件不存在导致 500。

**修复**：将 SPA 回退改为 `try-catch` 机制，当静态文件不存在时内联返回基础 HTML 模板。

### 2. 修复文章内链接跳转异常

**问题**：文章内的外部链接被重写为 `<a href="/link?url=...">`，在 Tauri 环境中点击后触发了完整页面跳转至 `http://tauri.localhost/link?url=...`，导致页面无法正常渲染。

**原因**：`rewriteLinks` 生成的 `<a>` 标签是原生 HTML 链接，在 Tauri 的 WebView 中点击会导致整页导航，而 Tauri 本地服务器没有 SPA 回退处理。

**修复**：在文章内容区添加点击事件拦截器，检测到 `/link?` 前缀的链接时，使用 React Router 的 `navigate()` 进行客户端路由，避免整页跳转。

### 3. 文章渲染策略优化

文章编辑器仅支持 Markdown 语法编写，但对已有的 HTML 格式文章保持完全向下兼容。通过 `looksLikeHTML()` 检测内容首字符是否为 `<` 来判断格式：HTML 内容透传，Markdown 内容按语法转换为 HTML。

---

## 技术优化

- 添加 `contentRef` / `id="article-content"` 用于链接点击拦截
- SPA 回退支持内联 HTML 模板
- Git 代理配置 `http://127.0.0.1:7897`
