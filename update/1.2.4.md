# SkyXing 1.2.4 更新日志

> 发布性质：功能完善 + Bug 修复（Feature + Bugfix）
> 覆盖平台：Web · Tauri APP (Windows/Android)

## 概要

1.2.4 在 1.2.3 基础上修复了主页无限刷新、Tauri 构建崩溃、OTA 缓存导致无法检测新版本等问题，新增全局刷新按钮，完善版本号动态注入和渠道分离机制。

---

## 新增功能

### 一、全局刷新按钮

每个页面右上角新增「刷新」按钮（🔄 图标），方便用户在有数据更新但自动同步尚未触发时强制刷新页面。支持桌面端和 Android 端。

### 二、OTA 缓存优化

- 缓存 TTL 从 300 秒缩短至 60 秒
- 新增 `?nocache=true` 查询参数，可强制绕过缓存实时拉取最新版本
- 应用首次检测更新失败时自动重试

### 三、正式版/测试版渠道分离

稳定版渠道仅检测正式发布版本，测试版渠道仅检测预发布版本（通过 GitHub `prerelease` 标志位 + 标签后缀 `-a`/`-b`/`-rc` 判断）。

---

## 问题修复

### 1. 修复主页无限刷新

`loadArticles` 改用 `useCallback` 包装保持引用稳定，防止 `useSync` Hook 的 `useEffect` 因函数引用变化不断重启导致无限循环。

### 2. 修复 Tauri 启动崩溃

`const loadArticles` 声明在 `useSync` 和 `useEffect` 引用之后，触发了 JavaScript TDZ。已调整声明顺序。

### 3. 修复版本号显示错误

「我的」页面底部版本号从硬编码 `1.2.1` 改为 Vite `define` 注入 `__APP_VERSION__`，自动跟随 `package.json`。

### 4. 修复 OTA 检测「No release found」

因 KV 缓存导致新发布的版本无法被即时检测到。缩短缓存 TTL 并新增 `nocache` 旁路参数。

---

## 技术优化

- Vite `define` 注入 `__APP_VERSION__`、`__APP_PLATFORM__`
- `useCallback` 包装数据加载函数，保持引用稳定
- `isPrerelease()` 函数准确判断发布类型
- Markdown 转换器重构，避免 Vite 压缩 TDZ 冲突
