# SkyXing 1.2.3 更新日志

> 发布性质：功能完善 + Bug 修复（Feature Enhancement + Bugfix）
> 覆盖平台：Web · Tauri APP (Windows/Android)

## 概要

1.2.3 在 1.2.2 基础上修复了 Tauri 生产构建崩溃问题，并完善了 Markdown 编辑器与外部链接安全跳转页面的细节。

---

## 新增功能

### 一、外部链接安全跳转

文章中的所有外部链接现在会自动经过 `/link?url=...` 安全跳转页，而非直接跳转。跳转页会：

- 显示目标链接的完整 URL 和域名
- 展示安全风险提醒（非官方域名、防范钓鱼等）
- 用户需点击「继续访问」才在新标签页打开链接

**覆盖范围**：Web / Tauri APP 两端均已实现。无论作者用 Markdown 语法 `[链接](url)` 还是原生 HTML `<a href="...">` 编写，均生效。

### 二、Markdown 编辑器完善

写文章编辑器新增 Markdown 语法支持，可与原生 HTML 混合编写。支持语法包括：

- 标题 `# ~ ######`
- 加粗 `**text**`、斜体 `*text*`、删除线 `~~text~~`
- 行内代码 `` `code` ``、代码块 ` ``` `（可指定语言）
- 链接 `[text](url)`、图片 `![alt](url)`
- 无序列表 `- item`、有序列表 `1. item`
- 引用 `> quote`、分割线 `---`

编辑器会智能检测内容格式：以 `<` 开头的视为 HTML 原样透传，否则按 Markdown 解析后转为 HTML。

### 三、文章置顶功能

管理员可在文章详情页对任意文章执行置顶/取消置顶操作。置顶文章优先显示在列表顶部，并带有「📌 置顶」蓝色标签。

- API：`PUT /articles/:id/pin`（需 admin 角色）
- 排序：置顶优先，同级按发布时间倒序
- 全端展示一致

### 四、评论置顶功能

文章作者可在评论区置顶高价值评论。置顶评论以蓝色左边框 + 浅蓝背景高亮展示，并优先排序。

- API：`PUT /comments/:id/pin`（需文章作者或 admin 角色）
- 排序：置顶优先，同级按发布时间正序

### 五、实时数据同步

全端引入 `useSync` Hook，基于轻量版本号轮询实现数据同步：

- 每 30 秒调用 `GET /state/version` 检查版本号
- 页面从后台切回前台时立即触发检查
- 标签筛选或搜索激活时自动暂停轮询
- 版本号变化时自动重新拉取文章列表

## Bug 修复

### 1. 修复 Tauri 应用启动崩溃（严重）

**问题**：1.2.2 版本构建的 Tauri APP（Windows 和 Android）启动后立即白屏，控制台报错：

```
ReferenceError: Cannot access 'I' before initialization
    at Og (index-DhLo6SPd.js:240:9498)
```

**原因**：`markdown.js` 中多个 `export function` 导出与内部函数的 `const` 变量引用，在 Vite 的 esbuild 压缩阶段产生了 Temporal Dead Zone（TDZ）冲突。压缩后变量 `I`（原变量经压缩重命名）被访问时尚未初始化。

**修复**：重构 `markdown.js`，将所有内部辅助函数（`mdToHtml`、`rewriteLinks`、`looksLikeHTML`）改为私有函数而非导出，仅保留 `prepareArticleContent` 一个导出入口。同时改用 `\x00` 哨兵字符替代文本占位符，避免与正文内容冲突。

### 2. 修复文章创建时缺少 `pinned` 字段

新建文章的数据结构中缺少 `pinned: false` 默认值，导致文章列表排序时该字段为 `undefined`。后端 `POST /articles` 处已补充默认值。

### 3. 修复评论创建时缺少 `pinned` 字段

与上述相同，评论创建时补充 `pinned: false` 默认值。

### 4. 修复评论列表缺少置顶排序

评论列表原按 `createdAt` 正序排列，未将置顶评论优先。已改为先按 `pinned` 降序、再按 `createdAt` 正序排列。

### 5. 修复无法通过用户名查询用户

新增端点 `GET /server/api/lookup/:username`，通过 `USERNAME_INDEX`（`username:xxx` → userId）直接查询用户信息。此前尝试在 `users` 子路由器下添加 `/by-username/:username` 路由，但与 `/:id` 路由冲突导致 404。

### 6. 修复 APP 端文章内容安全漏洞

`app/src/pages/ArticlePage.jsx` 中直接使用 `dangerouslySetInnerHTML={{__html: article.content}}` 渲染文章内容，未经过 `sanitizeHTML` 过滤，存在 XSS 风险。现已改为先通过 `prepareArticleContent` 转换（Markdown → HTML + 链接重写），再由外层 `sanitizeHTML` 过滤后渲染。

## 技术优化

- `markdown.js` 精简重构：内部函数私有化，代码体积缩小，避免构建工具压缩冲突
- 所有 API 客户端同步新增 `pinArticle`、`pinComment`、`getUserByUsername`、`getStateVersion` 四个方法
- 跨端置顶 UI 统一：Web 使用 `lucide-react` Pin 图标，移动端使用文本标签
- 引入 `tools/promote-admin.mjs` 管理脚本，支持通过 wrangler kv 直接提权
- 外部链接正则兼容性优化：确保 `http://` 和 `https://` 格式均能正确匹配重写

## 构建说明

### Tauri APP Windows 桌面端
```bash
cd app
npm run tauri:build:windows
```
产物：`app/src-tauri/target/x86_64-pc-windows-msvc/release/bundle/nsis/SkyXing_1.2.3_x64-setup.exe`

### Tauri APP Android
```bash
cd app
npm run tauri:build:android
```
产物：`app/src-tauri/gen/android/app/build/outputs/apk/universal/release/app-universal-release.apk`

### 后端部署
```bash
cd <server-root>
npx wrangler deploy
```
