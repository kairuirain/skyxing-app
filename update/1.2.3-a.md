# SkyXing 1.2.3-a 更新日志

> 发布性质：测试版（Beta for v1.2.4）
> 覆盖平台：Web · Tauri APP (Windows/Android)

## 概要

1.2.3-a 是 v1.2.4 正式版发布前的测试版本，主要包含版本号动态注入、OTA 双渠道分离、主页无限刷新修复，以及外部链接安全跳转等新特性。

---

## 新增功能

### 一、版本号动态注入

此前 APP 内的版本号（如「我的」页面底部的 `v1.2.1`）是硬编码在源代码中的，每次更新都需要手动修改多个文件，且经常与实际构建版本不一致。

现在通过 Vite `define` 在构建时自动从 `package.json` 读取版本号并注入：

- `__APP_VERSION__`：自动跟随 `package.json` 中的 `version` 字段
- `__APP_PLATFORM__`：固定为 `app`
- OTA 更新检测通知中的版本号也自动跟随，不再硬编码

### 二、OTA 正式版/测试版渠道分离

更新设置页面中的「稳定版/测试版」切换开关现在真正生效：

- **稳定版渠道**：仅检测 GitHub 上标记为正式版（非 Pre-release）的发布
- **测试版渠道**：仅检测 GitHub 上标记为 Pre-release 的发布（或标签名包含 `-a`、`-b`、`-rc` 等后缀）

此前的实现中，测试版渠道会错误地返回最新正式版，导致用户在测试版通道收到正式版更新提示。本次重构了 `pickRelease` 函数，通过 `isPrerelease()` 准确判断发布类型。

### 三、外部链接安全跳转页

文章中的所有外部链接（非 SkyXing 域名）会自动经过 `/link?url=...` 安全跳转页：

- 显示目标链接的完整 URL 和域名
- 展示安全风险提示：「非 SkyXing 官方域名」「请勿泄露账号密码」等
- 用户需点击「继续访问」按钮后才在新标签页打开

支持 Markdown `[链接](url)` 语法和原生 HTML `<a href="...">` 两种写法。

### 四、Markdown 编辑器

写文章页面新增 Markdown 语法支持，可与原生 HTML 混合编写：

- 标题 `# ~ ######`
- 加粗 `**text**`、斜体 `*text*`、删除线 `~~text~~`
- 行内代码 `` `code` ``、代码块 ` ``` `（可指定语言）
- 链接 `[text](url)`、图片 `![alt](url)`
- 列表、引用、分割线

## 问题修复

### 1. 修复主页无限刷新

**问题**：进入首页后文章列表每隔几秒刷新一次，页面闪烁无法正常使用。

**原因**：`loadArticles` 是组件内的普通 `const` 函数，React 每次渲染都重新创建函数引用。`useSync` 将其作为依赖，每次渲染都检测到引用变化 → `useEffect` 重新执行 → 调用 `loadArticles()` → `setState` 触发又一次渲染 → 死循环。

**修复**：`loadArticles` 和 `loadTags` 改用 `useCallback` 包装，保持引用稳定。

**影响平台**：Web / Tauri APP / Windows / Android

### 2. 修复版本号显示错误

**问题**：安装新版本后「我的」页面底部仍显示旧版本号 `v1.2.1`。

**原因**：版本号在 `MinePage.jsx` 中写死为 `'1.2.1'`，从未随构建更新。

**修复**：改为从 Vite `__APP_VERSION__` 注入，自动跟随 `package.json`。

### 3. 修复 APP 端文章内容安全漏洞

`app/src/pages/ArticlePage.jsx` 中直接使用 `dangerouslySetInnerHTML` 渲染文章内容，未经过 `sanitizeHTML` 过滤。现已改为先通过 `prepareArticleContent` 转换（Markdown→HTML + 链接重写），再由 `sanitizeHTML` 过滤后渲染。
