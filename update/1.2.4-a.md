# SkyXing 1.2.4-a 更新日志

> 发布性质：测试版（Beta for v1.2.4）
> 覆盖平台：Web · Tauri APP (Windows/Android)

## 概要

1.2.4-a 是 v1.2.4 正式版的测试预览版本，专注于 OTA 更新机制的稳定性和可靠性优化。

---

## 新增功能

### 一、OTA 缓存旁路机制

发布 `v1.2.4-a` 等新版本后，客户端可能因 Cloudflare KV 缓存（默认 5 分钟）而无法立即检测到新版本提示「No release found」。

现增加 `?nocache=true` 查询参数支持，强制绕过缓存实时拉取 GitHub Releases API：

```
GET /server/api/updates/check?platform=app&current=1.2.3-a&channel=beta&nocache=true
```

应用首次检测更新失败时，可自动重试一次带 `nocache=true` 的请求以获取最新数据。

### 二、缓存 TTL 缩短

默认 OTA 缓存时间从 300 秒缩短至 60 秒。开发测试期间新发布可在 1 分钟内被检测到，无需等待 5 分钟。

生产环境可通过 Cloudflare Dashboard 调整 `update:config` KV 项的 `cacheTtl` 字段恢复至 5 分钟。

## 问题修复

### 1. 修复 OTA 检测到「No release found」错误

**问题**：在 GitHub 上发布新的预发布版本（如 `v1.2.4-a`）后，测试版渠道客户端即使更新通道切换到「测试版」，仍提示「No release found」。

**原因**：Cloudflare Worker 缓存了 5 分钟的 GitHub Releases 列表，新发布在缓存期内无法被检测到。

**修复**：
- 缩短缓存 TTL 至 60 秒
- 增加 `?nocache=true` 旁路参数

### 2. 修复正式版/测试版渠道未真正分离

**问题**：`pickRelease` 函数在测试版渠道错误地返回最新正式版。

**修复**：通过 `isPrerelease()` 函数准确判断发布类型（GitHub `prerelease` 标志位 + 标签名后缀 `-a`/`-b`/`-rc`），稳定版渠道只返回正式版、测试版渠道只返回预发布版。
