# SkyXing 1.2.2 更新日志

> 发布性质：置顶系统 + 实时同步 + Markdown 编辑器 + 外部链接安全跳转
> 覆盖平台：Web · Tauri APP (Windows/Android) · React Native Android · Electron Windows

1.2.2 是一次大版本更新，引入置顶系统（文章置顶 + 评论置顶）、全端实时数据同步机制、Markdown 编辑器支持，以及外部链接安全跳转页。

## 一、文章置顶功能

- **管理员专属**：仅 `admin` 角色的用户可对任意文章执行置顶/取消置顶操作。
- **排序规则**：置顶文章优先排列在列表顶部，置顶文章之间按发布时间倒序排列。
- **前端标识**：所有平台的文章卡片均显示「📌 置顶」标签。
- **API 端点**：`PUT /articles/:id/pin`（管理员权限），切换文章的 `pinned` 布尔值。

## 二、评论置顶功能

- **文章作者专属**：仅该篇文章的作者（`authorId` 匹配）可对评论执行置顶/取消置顶操作，管理员也可操作。
- **排序规则**：置顶评论优先排列在评论区顶部。
- **前端标识**：置顶评论以高亮背景 + 蓝色左边框 + 「置顶」标签标识。
- **API 端点**：`PUT /comments/:id/pin`（需登录且为文章作者或管理员），切换评论的 `pinned` 布尔值。

## 三、管理员提权支持

- **新增端点**：`GET /server/api/lookup/:username`，通过用户名查询用户 ID 及基本信息。
- **提权流程**：管理员先通过用户名查得用户 ID，再调用 `PUT /admin/users/:id/role` 将角色改为 `admin`。
- **一键脚本**：内置 `tools/promote-admin.mjs`，通过 wrangler kv:key 直接修改 KV。

## 四、实时数据同步

- **轻量版本号机制**：`GET /state/version` 返回当前时间戳版本号。
- **全端轮询**：Web / Tauri APP / Windows / Android 统一引入 `useSync` Hook，每 30 秒自动检查版本号变化。
- **可见性检测**：页面从后台切回前台时立即触发版本检查。
- **智能跳过**：标签筛选或搜索激活时不轮询，避免干扰筛选结果。
- **触发刷新**：版本号变化时自动重新拉取文章列表。

## 五、Markdown 编辑器

- **混合编写**：编辑器同时支持 Markdown 语法和原生 HTML 标签，可混写。
- **支持语法**：`# 标题`、`**加粗**`、`*斜体*`、`` `代码` ``、`[链接](url)`、`![图片](url)`、`---` 分割线、`> 引用`、列表等。
- **安全过滤**：所有内容经过白名单 HTML 净化器（sanitize）过滤，移除危险标签和属性。
- **向后兼容**：已有 HTML 文章不受影响，Markdown 渲染在文章展示时按需转换。

## 六、外部链接安全跳转

- **跳转页机制**：文章中所有外部链接自动经过 `/link?url=...` 安全跳转页。
- **风险提醒**：跳转页显示目标域名和风险提示，提醒用户注意钓鱼风险。
- **确认操作**：用户需点击「继续访问」按钮才会在新标签页打开目标链接。
- **API 端点**：`GET /server/api/lookup/:username` — 用户查询端点。

## 七、修复问题

- 修复文章创建时缺少 `pinned` 字段的问题（默认 `false`）。
- 修复评论创建时缺少 `pinned` 字段的问题（默认 `false`）。
- 修复评论列表缺少置顶排序的问题。
- 修复无法通过用户名查询用户信息的问题（Hono 路由冲突导致 404）。
- 修复 APP 端文章内容未经 `sanitizeHTML` 过滤的安全漏洞。

## 八、技术优化

- 文章/评论列表排序统一为「置顶优先 + 时间倒序」。
- 所有 API 客户端同步新增 `pinArticle`、`pinComment`、`getUserByUsername`、`getStateVersion` 四个方法。
- 跨端置顶 UI 统一：Web 使用 `lucide-react` Pin 图标，移动端使用文本标签。
- 引入 `tools/promote-admin.mjs` 管理脚本，支持通过 wrangler kv 直接提权。
