# SkyXing 1.2.0 更新日志

> 发布性质：重大更新——自动更新系统、Web 导航重构与全新 OTA 系统（Major Release: Auto-Updates, Web Redesign & New OTA System）
> 覆盖平台：Web · Android · Windows

1.2.0 是自研 OTA 更新系统的首个版本，支持应用内自动检测更新与一键下载。同时 Web 端完成导航重构，提供更清晰的内容组织与跨端一致的体验。

## 一、全新 OTA 自动更新系统

- **自研更新引擎**：抛弃 Tauri 内置 Updater（数字签名过于复杂），采用兼容旧版 OTA 的自研方案。
- **B站风格提示界面**：检测到新版本时弹出粉蓝渐变更新弹窗，进度条显示下载状态，支持「立即更新」与「稍后再说」。
- **动态检测**：调用 `https://skyxing.dpdns.org/server/api/updates/check?platform={客户端}&current={版本}&channel={通道}` 获取更新数据。
- **GitHub Release 联动**：后端自动查询 GitHub Releases 获取最新版本与下载链接，发布即生效。
- **全端兼容**：无缝支持 1.1.x 旧版本的 OTA 端点，升级透明。

## 二、Web 端导航重构

- **五 Tab 导航**：主页 | 博客 | 私信 | 下载 | 我的，桌面端顶部水平导航条 + 移动端底部导航。
- **主页**：平台介绍 Hero + 文章列表（分页+标签筛选）+ 消息通知侧栏（登录后显示未读计数与最近会话）。
- **博客页**（/blog）：专注文章内容的干净列表页，无 Hero 无侧栏。
- **下载页**（/download）：版本号与下载链接**动态获取**，与 GitHub Release 自动同步，无需手动更新。
- **我的页**（/me）：个人卡片（头像/用户名/角色）、快捷操作（文章管理/评论/私信/设置）、管理员入口、退出登录。
- **设置页精简**：仅保留调试诊断（用户 ID / 角色 / JWT / 登录状态）。

## 三、实时通信优化

- **智能轮询**：3 秒轻量心跳 `unread-count`，仅 `latestUpdatedAt` 变化时加载全量。
- **Visibility API 暂停**：标签页不可见时暂停所有轮询，切回即查——零无用请求。
- **手动刷新按钮**：私信列表与对话详情页带旋转动画的刷新按钮。
- **消息同步修复**：header 显示对方用户名、4 秒轮询 + 乐观更新（发送后立即追加）。

## 四、性能优化

- **列表并行获取**：文章列表/标签/评论 `kvGetMany` 并行，消除串行 KV 往返。
- **评论索引 O(1)**：`COMMENT_INDEX:<articleId>` 取代全量扫描，旧数据首访自动懒重建。
- **边缘缓存头**：文章列表（`max-age=60`）、标签（`max-age=300`）、评论（`max-age=30`），Cloudflare 边缘缓存降低源站压力。

## 五、安全加固

- **JWT_SECRET 外置**：从配置明文移除，改为 Cloudflare 加密 Secret 注入。
- **密码哈希 PBKDF2**：旧 SHA-256 升级为 PBKDF2-HMAC-SHA256（10 万次迭代 + 随机盐），登录自动迁移。
- **注册 O(1) 去重**：邮箱索引替代 `kvList` 全量扫描。

## 六、修复问题

- 修复 JDK 26 导致 Android 构建失败的问题。
- 修复 `@tauri-apps/plugin-shell` 缺失导致 Tauri 前端构建失败的问题。
- 修复评论索引 `parseJson` 错误导致 500 的问题。
- 修复消息 header 显示用户 ID 而非用户名的问题。
- 修复消息发送后对方无法实时同步的问题。
