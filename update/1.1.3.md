# SkyXing 1.1.3 更新日志

> 发布性质：发布流程规范化与体验修复（Release Pipeline & UX Fixes）
> 覆盖平台：Web · Android · Windows

1.1.3 在 1.1.2 基础上进一步规范化全平台发布构建流程，并修复若干影响日常使用的体验问题。

## 一、新增功能
- 增加独立更新日志目录与按版本号归档的发布说明（`./update/[版本号].md`），便于用户追溯每个版本的变更。
- Android 客户端支持以 universal APK 形式发布，简化不同架构设备的分发。

## 二、改进项
- 统一并固化 Android 构建所需的环境要求（JDK 17 + 指定 NDK 版本），避免构建环境差异导致的失败。
- Windows 安装包构建产物集中归档至 `releases/Windows`，发布目录结构更清晰（已加入版本忽略，不随仓库提交）。
- 优化多端版本号来源，版本定义集中管理，降低遗漏与不一致风险。
- 进一步收敛构建输出与本地密钥（签名文件等）的提交范围，保持仓库整洁。

## 三、修复问题
- 修复 Android 构建在默认 Java 版本过高（不兼容 Gradle）时的失败，明确建议使用 JDK 17。
- 修复 Windows 客户端安装后版本信息显示滞后的问题。
- 修复若干边缘场景下文章富文本与评论的渲染错位。
- 修复管理后台在大数据量分页下的加载卡顿。
