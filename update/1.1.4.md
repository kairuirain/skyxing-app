# SkyXing 1.1.4 更新日志

> 发布性质：安全加固、性能优化与 OTA 全端支持（Security & Performance & OTA）
> 覆盖平台：Web · Android · Windows · Tauri 桌面端

1.1.4 是一次深度的安全与性能升级，同时补全了 OTA 更新功能的全端体验，并完成了构建流程的完整验证与发布自动化。

## 一、安全加固

- **JWT_SECRET 外置**：不再以明文写入 `wrangler.toml`，改为通过 Cloudflare Workers 加密 Secret 注入，避免密钥随代码泄露至仓库。
- **密码哈希升级**：从旧版静态盐 SHA-256 升级为 PBKDF2-HMAC-SHA256（10 万次迭代 + 每用户随机 16 字节盐），格式 `pbkdf2$iter$salt$hash`。旧格式登录后自动迁移，透明无感。
- **注册邮箱 O(1) 去重**：建立 `EMAIL_INDEX` 索引，替代原先的 `kvList` 全量扫描，避免邮箱被枚举。

## 二、性能优化

- **列表并行批量获取**：文章列表/标签/评论读取由串行逐个 `kvGet` 改为 `kvGetMany`（`Promise.all` 并行），消除 N 次串行 KV 往返。
- **评论按文章索引（O(1)）**：新增 `COMMENT_INDEX:<articleId>` 索引，评论列表从全量 `kvList` 扫描改为直接读索引。旧数据通过首次访问自动懒重建索引，零数据丢失。
- **公开只读接口边缘缓存**：文章列表（`max-age=60`）、标签（`max-age=300`）、评论（`max-age=30`）增加 `Cache-Control` 头，由 Cloudflare 边缘缓存，大幅降低源站压力。

## 三、OTA 全端更新支持

- **OTA 后端升级**：新增 `/updates/check`、`/updates/latest`、`/updates/config` 接口，支持 stable / beta 更新通道、基于 `cf-ipcountry` 的 GitHub 代理路由（国内走 ghtfast.top）。
- **四端设置页更新 UI**：Web、Windows（Electron）、Android（Expo）、Tauri 桌面端均新增"调试 / 更新 / 关于软件"三段式设置页，集成 OTA 检测与下载。
- **Python 发布管理器**：`tools/release_manager.py`（纯标准库），支持配置文件、dry-run 预演、强制覆盖，一行命令完成 Release 创建与资源上传。

## 四、代码规范化

- **统一鉴权中间件**：`articles.js` 复用 `rbac.js` 的共享 `authRequired`，移除重复本地实现。
- **清理无用导入**：移除已不再使用的 `kvList` 等导入。
- 各端 `api.js` 方法保持一致，为后续跨目录共享打下基础。

## 五、修复问题

- 修复 JDK 版本过高（JDK 24+）导致的 Gradle 构建失败，规范推荐使用 JDK 17。
- 修复评论列表 `parseJson=false` 导致索引读回字符串而非数组、第二次请求返回 500 的问题。
- 修复 `@tauri-apps/plugin-shell` 依赖缺失导致 Tauri 前端构建失败的问题。
- 修复评论删除后索引未同步更新、再次查询可能出现脏数据的问题。
