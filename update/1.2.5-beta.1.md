# SkyXing 1.2.5-beta.1 更新日志

> 发布性质：测试版（Beta for v1.2.5）
> 覆盖平台：Web · Tauri APP (Windows/Android)

## 概要

1.2.5-beta.1 是 v1.2.5 正式版发布前的测试版本，主要修复了 Web 端路由刷新 500 错误、Tauri 端文章链接跳转异常、App 主页无限刷新等问题，并新增 Android 默认浏览器打开链接支持。

---

## 问题修复

### 1. 修复 Web 端路由刷新 500 错误

**问题**：主页 `/` 刷新正常，但在 `/article/xxx`、`/write` 等页面刷新或直接访问时返回 `Internal Server Error`。

**原因**：Cloudflare Worker 的 SPA 回退 `serveStatic` 因静态资源文件不可访问导致 500。

**修复**：将 SPA 回退改为硬编码内联 HTML，确保所有路径均可正常加载前端应用。

### 2. 修复文章内链接跳转异常

**问题**：文章外部链接被重写为 `<a href="/link?url=...">` 后，在 Tauri 环境中点击触发了整页跳转至 `http://tauri.localhost/link?url=...`，导致白屏。

**修复**：
- 文章内容区添加点击拦截器，检测到 `/link?` 链接时使用 React Router 客户端导航
- 跳转页「继续访问」按钮改为 `@tauri-apps/plugin-opener.openUrl()`，通过系统默认浏览器打开

### 3. 修复 App 主页无限刷新

**问题**：进入首页后文章列表频繁刷新闪烁。

**原因**：`loadArticles` 是普通 `const`，每次渲染重新创建 → `useSync` 的 `useEffect` 因依赖引用变化不断重启 → 死循环。

**修复**：`loadArticles` 和 `loadTags` 改用 `useCallback` 包装，保持引用稳定。

### 4. 修复文章页 React 报错 #310

**问题**：打开任意文章页显示空白，控制台报 `Minified React error #310`。

**原因**：`useEffect`（链接拦截器）声明在了早期 `return` 语句之后，导致 hooks 数量不一致。

**修复**：将 `useEffect` 移到所有早期 `return` 之前。

### 5. 修复 Android 默认浏览器打开链接

**问题**：Android 端点击外部链接无法通过系统浏览器打开。

**修复**：
- 安装 `@tauri-apps/plugin-opener` 插件
- 注册 `tauri_plugin_opener::init()` 到 Rust 端
- 添加 `opener:default` 权限
- `openExternal` 和「继续访问」按钮均改用 `openUrl()` 调用

## 新增功能

### OTA 渠道切换确认弹窗

当 App 在测试版通道检测到正式版更新（或反之），会弹出确认弹窗提示「您将从 A 切换到 B 渠道」，用户确认后才显示更新详情。

### 版本号系统

正式版 `MAJOR.MINOR.PATCH`（如 `1.2.5`），测试版 `MAJOR.MINOR.PATCH-stage.N`（如 `1.2.5-beta.1`）。
